## Key Decisions Made

**Architecture:**
1. **Manual Refresh Only for Celery Monitoring**
   - NO auto-polling on Celery endpoints (they take 4s each)
   - User clicks "Refresh" to load data
   - Prevents reintroducing circular dependency/performance issues
   - All hooks use `enabled: false` by default

2. **Fixed Circular Health Check**
   - SSE stream was calling `check_backend_api()` which called `/health`
   - Created circular dependency: SSE → health → service check → HTTP to self
   - Fixed: Added `skip_http_check` parameter, SSE now skips slow checks

3. **Memoryview Handling**
   - PostgreSQL returns result/traceback as memoryview objects
   - Must convert to strings using `.tobytes().decode("utf-8")`
   - Added proper error handling for conversion

**Patterns Established:**
- Service monitoring with `skip_slow_checks` parameter
- Manual refresh UI pattern for slow operations
- Clear user messaging about performance tradeoffs
