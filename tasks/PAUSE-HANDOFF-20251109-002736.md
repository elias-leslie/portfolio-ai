## Key Decisions

**Architecture:**
1. **Portfolio-Watchlist Sync**: One-way, additive only (portfolio → watchlist)
   - No cascade deletes between portfolio and watchlist
   - Users maintain full control (can remove portfolio-sourced items)
   - Sync happens automatically on every portfolio API fetch

2. **Portfolio Badge Logic**: Check current portfolio state, not historical source
   - Initially: Badge showed based on `source='portfolio'` (historical)
   - Fixed: Badge checks if symbol currently in portfolio positions
   - Result: Accurate "Portfolio" badge reflecting real-time state

3. **News Integration**: Dashboard-centric, removed standalone page
   - Market news now primary dashboard feature
   - Eliminated redundant /news route
   - Reduced navigation complexity

**Patterns Established:**
- Auto-sync on data fetch (portfolio API)
- Historical metadata (source field) vs current state (badge display)
- React hooks for portfolio data fetching in watchlist
- Color-coded rows for gain/loss visualization

**Issues Discovered & Fixed:**
1. **None values in priority calculations** - Added filtering in priority.py
2. **Directory permissions** - Fixed components/dashboard to 755
3. **Missing date-fns dependency** - Installed via npm
4. **Badge showing for removed positions** - Fixed to check current portfolio

**Band-Aids Avoided:**
- ✅ No hardcoded portfolio symbols
- ✅ No duplicate source tracking logic
- ✅ No workarounds for None values (fixed root cause)
- ✅ No manual badge management (automatic via portfolio state)
