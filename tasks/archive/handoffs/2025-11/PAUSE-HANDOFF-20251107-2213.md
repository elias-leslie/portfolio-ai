## Key Discoveries & Decisions

### Critical Bug Found
**CASCADE Delete Data Loss**:
- `watchlist_items.account_id` had `ON DELETE CASCADE`
- User deleted test portfolio accounts → CASCADE deleted entire watchlist (24 items)
- All snapshots also deleted (CASCADE on snapshots table too)

**Immediate Fix Applied**:
- Changed FK to `ON DELETE RESTRICT`
- Restored watchlist items from `news_cache` table (unique tickers)
- Recreated "default" portfolio account

**Root Cause Analysis**:
- Watchlist was designed with `account_id` for future multi-user feature
- Frontend hardcodes `accountId = "default"` with TODO comment
- Currently watchlist is effectively global, not per-account
- **Confusion**: Watchlist = "stocks I'm monitoring" vs Positions = "stocks I own"
- Only POSITIONS should be tied to accounts, not watchlist

### Architecture Decisions Made

**Service Management**:
- ✅ Confirmed Claude CAN restart services autonomously via scripts
- ✅ Scripts work because sudoers allows specific systemctl commands via NOPASSWD
- ❌ Direct `sudo` commands still require TTY (expected behavior)
- Documentation updated to clarify capabilities

**React Query Cache**:
- Fixed invalidation with `refetchType: 'active'`
- Applied to both watchlist and portfolio mutations
- Ensures UI updates immediately after mutations

**Foreign Key Strategy**:
- Changed from CASCADE to RESTRICT for safety
- Prevents accidental data loss from deleting related records
- Forces explicit handling of relationships

## Issues Discovered (Deferred to Next Session)

### 1. Portfolio UI Design Issue
**Current**: Accounts and positions shown as separate cards
**Expected**: Accounts with expandable/collapsible positions underneath

**User's Intent**:
- Add accounts first (e.g., "My IRA", "Taxable Account")
- Add positions under that account
- Expand/collapse to show/hide positions

### 2. Watchlist/Portfolio Data Model Confusion
**Problem**: `watchlist_items.account_id` shouldn't exist (or should be nullable/ignored)

**Correct Separation**:
- **Watchlist**: Stocks being monitored (global, not owned)
  - Should NOT be tied to portfolio accounts
  - Used for tracking potential investments
- **Portfolio Positions**: Stocks actually owned in specific accounts
  - MUST be tied to portfolio accounts
  - Different accounts have different positions

**Current State**:
- Frontend hardcodes `accountId = "default"`
- Watchlist treated as global despite having account_id FK
- No practical use of per-account watchlists

### 3. News Page Missing Portfolio Filter
**Current**: Market / My Watchlist toggle
**Needed**: Market / My Watchlist / **My Portfolio** (new)

Should show news for stocks actually owned in portfolio accounts.
